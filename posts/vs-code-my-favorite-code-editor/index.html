Visual Studio Code - My favorite code editor

January 22, 2018

Everything below about VS Code are my opinions & all about personal preference.

Yes! Visual Studio Code (VS Code) is my favorite & to go code editor. I have used Sublime Text & Atom too. I was great fan of Atom until I used VS Code. I have been using VS Code for almost a year now. I recently did give Atom (Version 1.23.3) a try again but there was still a huge performance difference when compared to VS Code. They both use Electron under the hood. Of course I need not mention that Sublime Text is the fastest of the three since it is based on C++.

BUT personally Sublime Text isn’t good enough out of the box. It needs tons of plugins for almost anything. Whereas both Atom & VS Code support a lot out of the box. At least the basic features for writing code, development, debugging & even Git.

From the official FAQ:

“Visual Studio Code is a streamlined code editor with support for development operations like debugging, task running and version control.”

And yeah, it is a code editor & NOT an Integrated Development Environment (IDE). It just has the basics which other editors don’t. I do use IDEs like Eclipse, RubyMine & PyCharm now & then but it just depends on the project. Anything front-end & I go to VS Code right away.

Some VS Code out of the box features that are awesome:

  • IntelliSense -> Syntax highlighting, autocomplete, smart completion based on variable types, function definitions & import modules.
  • Debugging
  • Git

VS Code has great TypeScript & JavaScript support out of the box without having to install any extensions. However, I do prefer using ESLint & TSLint & have my own configurations for linting & debugging. For me, VS Code has it all. It has the right performance for the best features & extensibility.
