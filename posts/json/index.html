JSON cheat sheet

January 8, 2018

JSON (Javascript Object notation) is a syntax & a standard for storing & exchanging data, specifically between the server & the client (like a browser or the front-end). JSON is “self-defined” or say “self-described”. It is easy to understand & learn. The JSON format was first specified by Douglas Crockford. It was extended from the Javascript scripting language. JSON is language independent.

Every JSON file has the extension .json & the internet media type/Multipurpose Internet Mail Extensions (MIME) type is application/json while the uniform type identifier is public.json.

Example json:

{
   "Computer Science & Engineering": [
      {
         "Subject":"Data Structures",
         "language": "Python, Java, Ruby",
         "edition": 3,
         "author": "FirstName1 LastName1"
      },
      {
         "Subject":"Algorithms",
         "language": "Python, Java, Ruby",
         "edition": 2,
         "author": "FirstName2 LastName2"
      }
   ]
}

JSON syntax:

  • Data is name/value pairs.
  • Data is separated by commas.
  • Curly braces hold objects.
  • Square brackets hold arrays.
  • JSON supports two data structures:
    • Collection of name/value pairs.
    • Ordered list of values (array, list, vector or sequence etc.).

JSON data types:

  • String - must be written in double quotes.

Example: {"name":"Vanessa Ives"}

  • Number - integers or floating points.

Example: {"age": 33}

  • Boolean - true or false.

Example: {"like": true}

  • Array - enclosed within square brackets.

Example: {"characters": ["James Keziah Delaney", "Vanessa Ives"]}

  • Object - unordered set of name/value pairs enclosed within curly braces. The keys must be strings & unique.

Example: {"dreamCouple": {"Him": "James Keziah Delaney", "Her": "Vanessa Ives"}}

  • null

Example: {"show": null}

Following values are invalid JSON data types:

  • undefined
  • date
  • function
